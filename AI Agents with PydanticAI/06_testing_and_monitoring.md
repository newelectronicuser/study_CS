# Testing and Monitoring

One of the biggest advantages of PydanticAI is how easy it makes testing AI agents, which are traditionally difficult to verify.

## 1. Testing with `TestModel`
PydanticAI provides a `TestModel` that you can use to simulate LLM responses without actually calling an API.

```python
from pydantic_ai.models.test import TestModel

def test_my_agent():
    # Mock the LLM to return a specific string
    model = TestModel(custom_result_text="The capital of France is Paris.")
    agent = Agent(model)
    
    result = agent.run_sync("What is the capital of France?")
    assert result.data == "The capital of France is Paris."
```

## 2. Testing Structured Results
You can mock complex Pydantic objects as results.

```python
from pydantic_ai.models.test import TestModel

def test_extraction_agent():
    model = TestModel(custom_result_args={'name': 'Alice', 'age': 30})
    agent = Agent(model, result_type=User)
    
    result = agent.run_sync("Extract user")
    assert result.data.name == "Alice"
```

## 3. Verifying Tool Usage
You can check if the agent called the correct tools with the expected arguments.

```python
def test_tool_call():
    model = TestModel()
    agent = Agent(model)
    
    @agent.tool
    def my_tool(x: int):
        return x * 2
        
    agent.run_sync("Use tool with 5")
    # Verify that the tool was indeed called by the LLM (represented by TestModel)
```

## 4. Why Mock the Model?
- **Speed**: Unit tests run in milliseconds instead of seconds.
- **Cost**: No API charges during local development or CI/CD.
- **Determinism**: Ensures your tests don't fail because the LLM decided to change its phrasing today.
- **Edge Cases**: Easily test how your agent handles bad inputs or empty tool results.

## 5. Monitoring and Logging
PydanticAI integrates with standard Python logging. You can set the log level to `DEBUG` to see:
- The exact prompt sent to the LLM.
- The raw JSON response from the LLM.
- Tool call details and result validation errors.

## 6. OpenTelemetry (Observability)
For production monitoring, you can use OpenTelemetry to track agent runs as spans. This allows you to visualize the "trace" of an agent run (Prompt -> Tool -> Search -> Result) in tools like Jaeger or Honeycomb.

> [!IMPORTANT]
> **Unit Test your Prompts?**: 
> You aren't really testing the "intelligence" of the LLM with `TestModel`. You are testing YOUR Python logic: "Did I pass the right dependencies?", "Is the result type correct?", "Does the tool handle the return value correctly?".

> [!TIP]
> **Snapshots**: 
> Use a library like `pytest-snapshot` or `syrupy` to save and verify the prompts generated by your dynamic system prompt functions.
