# Kanban and Lean

Kanban and Lean are methodologies that prioritize efficiency, flow, and the elimination of waste.

## 1. Kanban
Kanban is a visual system for managing work as it moves through a process. It focuses on Just-In-Time (JIT) delivery.

### Core Principles
- **Visualize Work**: Use a Kanban board (To Do, In Progress, Done) to see the state of every task.
- **Limit WIP (Work In Progress)**: Strictly limit the number of tasks in each column to identify bottlenecks and prevent multitasking.
- **Manage Flow**: Monitor how tasks move to ensure a smooth, continuous delivery.
- **Make Policies Explicit**: Clearly define how work is done (e.g., "Done" criteria).
- **Continuous Improvement (Kaizen)**: Use data to improve the process over time.

## 2. Lean Software Development
Derived from Toyota's Lean Manufacturing, it focuses on delivering value with minimum waste.

### The 7 Principles of Lean
1.  **Eliminate Waste**: Remove anything that does not add value (unused features, delays, defects).
2.  **Build Quality In**: Automate testing and stop the line when errors occur.
3.  **Create Knowledge**: Support learning and knowledge sharing within the team.
4.  **Defer Commitment**: Keep options open till the last possible moment (avoid locking in bad decisions).
5.  **Deliver Fast**: Ship software in small batches to get immediate feedback.
6.  **Respect People**: Empower the team to make decisions.
7.  **Optimize the Whole**: Look at the entire value chain, not just individual parts.

## 3. Kanban vs. Scrum

| Feature | Scrum | Kanban |
| :--- | :--- | :--- |
| **Cadence** | Fixed-length Sprints (Intervals) | Continuous Flow |
| **Commitment** | Team commits to Sprint Backlog | No fixed commitment (WIP limits) |
| **Roles** | Required (SM, PO, Dev Team) | No required roles |
| **Metrics** | Velocity (Points per sprint) | Cycle Time, Lead Time |

> [!TIP]
> **Scrumban** is a hybrid model that uses Scrum's structure (roles, events) with Kanban's visualization and WIP limits to manage workflow more flexibly.
