# SDLC Phases Overview

The Software Development Life Cycle (SDLC) is a structured process used by the software industry to design, develop, and test high-quality software.

## The 7 Core Phases

### 1. Planning (Requirement Collection)
The most important phase. Senior members of the team collect requirements from the client and stakeholders.
- **Key Deliverables**: Business Requirement Document (BRD), Feasibility Study.
- **Goal**: Define the project scope and identify resources.

### 2. Analysis (Defining Requirements)
In this phase, requirements are clearly defined and documented.
- **Key Deliverables**: Software Requirement Specification (SRS) document.
- **Goal**: Ensure everyone has a clear understanding of the project's functional and non-functional requirements.

### 3. Design
High-level and low-level architectural designs are created based on the SRS.
- **Key Deliverables**: Design Document Specification (DDS), System Architecture, Data Models.
- **Goal**: Define how the software will be built (Technology stack, database schema, API design).

### 4. Implementation (Coding/Development)
The actual coding takes place in this phase. Developers follow code guidelines and use tools like compilers, debuggers, and interpreters.
- **Goal**: Build the software according to the design specifications.

### 5. Testing
Once the code is developed, it's tested against the requirements to ensure that the product is actually solving the needs addressed and gathered during the requirements phase.
- **Goal**: Identify bugs and ensure quality. (Unit, Integration, and System testing).

### 6. Deployment
Once the software is tested and ready to go, it is released to the customers.
- **Goal**: Make the software available for end-users. (Beta testing, Staging, and Production).

### 7. Maintenance
Once customers start using the developed system, actual problems come up and need to be solved from time to time.
- **Goal**: Provide support, fix bugs, and add new features based on user feedback.

> [!IMPORTANT]
> The **SRS (Software Requirement Specification)** is the "single source of truth" that bridges the gap between business needs and technical implementation.
